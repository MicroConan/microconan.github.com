-如果希望static前缀的请求才去/public返回静态文件，则可以：

  • app.use方法是很重要的方法，当客户端过来一个请求时，它能按照你调用的顺序，调用这些方法，这些方法可以称之为中间件(middleware)。
  • 你可以根据文件后缀注册对应的模板引擎
  • express的模板缓存和所用模板引擎的缓存是不同的。
  • req.param是存储app.use(‘/user/:id’)这种形式里面req.param.id
  • req.query是存储/?name=conan这种形式req.query.name是conan
  • req.body是存储post请求里面，比如a=1
  • req.files是存储post上传的文件
  • req.cookies存储cookie键值对，由cookieParser中间件提供
  • req.signedCookies存储签名后的cookies键值对，由cookieParser中间件提供
  • req.path是端口之后?之前的字段
  • req.url则是端口之后所有字段
  • 无论req.path和req.url都是截取了app.use函数第一个参数之后的值
  • req.host请求的host不含端口号
  • req.fresh返回通过etag和Last-Modified比较是否相同
  • res.cookie(‘rememberme’, ‘1’, { expires: new Date(Date.now() + 900000), httpOnly: true });设置cookie
  • res.clearCookie(‘name’, { path: ‘/admin’ });
  • res.redirect([status], url)，重定向status默认302
  • res.charset = ‘value’;设置字符集
  • res.send(404, ‘Sorry, we cannot find that!’);发送响应
  • res.jsonp,获取query里面的callback的值返回jsonp
  • req.param(‘name’)，按照以下顺序查找
  • req.get(‘method’)，获取请求头里面的信息例如status、method、Cache-Control
  • req.accepts(types).检查给定的types 是不是可以接受类型，当可以接受时返回最匹配的，否则返回undefined - 这个时候你应该响应一个406 “Not Acceptable”.type 的值可能是单一的一个mime类型字符串,比如 “application/json”, 扩展名为”json”, 也可以为逗号分隔的列表或者数组。当给定的是数组或者列表，返回最佳匹配的。
  • req.accepted返回可以接受的类型从高质量到低质量

  • app.param(parameter, fn)是不好理解的一个点，它可以在你path匹配结果里面的参数做处理。
    举例来说用户删除id的地址是/user/delete/id，用户新增id的地址是user/add/id，然后注册分别是app.use('/user/delete/:id')和app.use('user/add/:id')，这时候他们都有需要去数据库获取id信息的一个业务操作，这时候我们可以把他们集中在一个函数里面操作，这时候可以”监听参数id”来实现了，就是当用户请求的path命中id参数时，我们可以调用个函数。这时候就可以用app.param(parameter,fn)了,参数parameter使用监听的参数(如id)，而fn则是要会掉的函数。我们可以把根据id来把用户信息获取出来保存在req里面给后面的app.use绑定的函数来处理。也就是说app.use可以绑定path回调，而app.param则可以绑定app.use的参数回调.

  • 谨慎在中间件使用setTimeout来操作，因为里面的错误无法被捕捉到
